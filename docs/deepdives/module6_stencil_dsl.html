üìã Overview

The Stencil DSL module enables users to define custom compute kernels at runtime using Lua scripts or C++ APIs. Shaders are automatically generated, compiled to SPIR-V, and cached to disk for fast reloading.

Key Features

  • Lua Scripting: Define stencils in pure Lua - no C++ recompilation needed
  • Automatic Code Generation: User writes simple macros, system generates full GLSL
  • Persistent Caching: SHA-256 hash-based SPIR-V cache (~500ms ‚Üí ~50ms)
  • Type Safety: Field names validated against FieldRegistry
  • Macro System: Read_field(idx) / Write_field(idx, val)

StencilParser

  • Parses Lua tables into StencilDefinition structs
  • Validates field names against FieldRegistry
  • Auto-detects neighbor radius and halo requirements
  • Comprehensive error messages

ShaderGenerator

  • Generates GLSL 4.6 compute shaders with BDA
  • Injects buffer references for all registered fields
  • Replaces Read_/Write_ macros with actual buffer accesses
  • Adds helper functions for neighbor access

PipelineCache

  • SHA-256 hashing of GLSL source for cache keys
  • Persistent disk storage in ~/.fluidloom/shader_cache
  • Automatic cache validation and miss handling
  • Thread-safe cache operations

Example 1: Simple Diffusion

-- diffusion.lua
local diffusion = {
    name = "thermal_diffusion",
    inputs = {"temperature"},
    outputs = {"temperature_new"},
    code = [[
        float T = Read_temperature(linearIdx);
        float alpha = 0.1;  // Diffusion coefficient
        
        // Simple explicit diffusion (Euler step)
        // In real use, would read neighbors
        float T_new = T * (1.0 - 6.0 * alpha) + alpha * 0.0; // Placeholder
        
        Write_temperature_new(linearIdx, T_new);
    ]],
    neighbor_radius = 1
}

return diffusion

üíª C++ Usage

Loading Lua Stencil

// Initialize Lua state
sol::state lua;
lua.open_libraries(sol::lib::base, sol::lib::math);

// Load stencil script
lua.script_file("diffusion.lua");
sol::table stencilTable = lua["diffusion"];

// Parse into C++ struct
StencilParser parser(fieldRegistry);
StencilDefinition def = parser.parse(stencilTable);

// Register and compile
StencilRegistry registry(context, fieldRegistry);
const auto& compiled = registry.registerStencil(def);

LOG_INFO("Stencil '{}' ready: {} pipeline", def.name, (void*)compiled.pipeline);

Direct C++ Definition

// No Lua needed - define in C++
StencilDefinition def{
    .name = "simple_add",
    .inputs = {"field_a", "field_b"},
    .outputs = {"field_c"},
    .code = R"(
        float a = Read_field_a(linearIdx);
        float b = Read_field_b(linearIdx);
        Write_field_c(linearIdx, a + b);
    )",
    .neighborRadius = 0
};

registry.registerStencil(def);

Dispatching Stencil

// Get compiled pipeline
const auto& stencil = registry.getStencil("thermal_diffusion");

// Record dispatch
vk::CommandBuffer cmd = ...;
cmd.bindPipeline(vk::PipelineBindPoint::eCompute, stencil.pipeline);

// Push constants with field addresses
struct PC {
    uint64_t field_temperature_addr;
    uint64_t field_temperature_new_addr;
    uint32_t activeVoxelCount;
} pc;

pc.field_temperature_addr = fieldRegistry.getField("temperature").deviceAddress;
pc.field_temperature_new_addr = fieldRegistry.getField("temperature_new").deviceAddress;
pc.activeVoxelCount = gridManager.getActiveVoxelCount();

cmd.pushConstants(stencil.layout, vk::ShaderStageFlagBits::eCompute, 0, sizeof(pc), &pc);

// Dispatch
uint32_t workgroups = (pc.activeVoxelCount + 127) / 128;
cmd.dispatch(workgroups, 1, 1);

üîç Generated GLSL Example

When you write this Lua:

code = [[
    float d = Read_density(linearIdx);
    vec3 v = Read_velocity(linearIdx);
    Write_density_new(linearIdx, d * 0.99);
]]

ShaderGenerator produces this GLSL:

#version 460
#extension GL_EXT_buffer_reference : require
#extension GL_EXT_buffer_reference2 : require
#extension GL_EXT_scalar_block_layout : require

layout(local_size_x = 128) in;

// Buffer references
layout(buffer_reference, scalar) buffer density_Buffer { float data[]; };
layout(buffer_reference, scalar) buffer velocity_Buffer { vec3 data[]; };
layout(buffer_reference, scalar) buffer density_new_Buffer { float data[]; };

// Push constants
layout(push_constant) uniform PC {
    uint64_t field_density_addr;
    uint64_t field_velocity_addr;
    uint64_t field_density_new_addr;
    uint32_t activeVoxelCount;
} pc;

void main() {
    uint linearIdx = gl_GlobalInvocationID.x;
    if (linearIdx >= pc.activeVoxelCount) return;
    
    // User code (macros replaced)
    float d = density_Buffer(pc.field_density_addr).data[linearIdx];
    vec3 v = velocity_Buffer(pc.field_velocity_addr).data[linearIdx];
    density_new_Buffer(pc.field_density_new_addr).data[linearIdx] = d * 0.99;
}
